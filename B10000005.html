Anti-Fraud Systems Data Processing

I will introduce how I use machine learning in the anti-fraud scenario. This article mainly focuses on data processing. As for training models and tuning parameters, they are still being explored. I will update the blog after making a breakthrough.

Background

Equipment such as mobile phones and computers plays a crucial role in the Internet risk control system. IP addresses and mobile phone numbers are valuable resources for operators, enabling them to gather essential risk attributes and data. On the other hand, users have direct control over their devices, allowing us to access various device information for effective risk control. The dimension of device data is extensive, and by implementing appropriate device idendify rules, we can greatly enhance protection measures. In order to counteract the risk control technology employed by equipment manufacturers, the illegal industry is continuously exploring innovative technologies, with machine-modification tools being particularly noteworthy.

The marketing activities of Internet business platforms typically involve the following customer terms that are commonly seen.

  • Only one participation is allowed per registered account.
  • Participation is restricted to a single occurrence per mobile phone number.
  • Each device can participate only once.

Certainly, these terms cannot solely depend on the customer's conscious compliance; they should be enforced through background limit policies. the illegal industry employs a limited number of authentic devices to generate an extensive array of counterfeit ones. How to identify the two participants are the same device is challenging.

Goals

The criminal group usually use the same device multiple times with modified data to pretend new devices, but not all of fields can be modified due to cost constraints. Therefore, by analyzing the reported data from each device, we can know what kind of report data are from the same device. We can use neural network to learn the pattern in our existing data, having the ability to identify devices, thereby preventing criminal groups from modifying device data for illegal purposes.

Data Processing
1. Data Source

The data for this business is stored in various forms and needs to be unified into a consistent format. Data sources include MaxCompute, Oss, HDFS, etc.

2. Business Fields

Data is a crucial foundation for machine learning tasks. It is necessary to select relevant data fields based on the business problem at hand. The algorithm's required data fields are determined based on user behavior in different scenarios. Data is collected from various sources, processed, associated, and integrated into a uniform format. For detecting fraudulent activities, device fingerprinting or other techniques are employed to gather device environment information, including mobile device brand, model, system, version, and network environment information (IP/gateway). Additionally, ID-type data such as IMEI, MAC, OAID, Android ID, Google ID, etc., are used to comprehensively cover device behavioral data from different dimensions.

3. Data Labels

Algorithms typically learn the patterns and characteristics of data from various perspectives. The presence and distribution of labeled samples in the dataset influence the choice of algorithmic models. If there are no labels, unsupervised models are used. If there are a few labels, semi-supervised models are considered. If there are a significant number of labels, supervised models are employed. In the case of supervised models, the algorithm learns from known label information, continuously optimizing the objective function to obtain an appropriate model, and eventually making predictions on test samples. In this algorithmic identification model, we use a large amount of historical data reported by clients for training.

4. Data Validation

Each field has specific standards, and over the years, we have accumulated various field value validation methods, mainly using regular expressions to determine whether a field complies with standards. For example, for the 'google_ads_id' field, a regular expression is used to check whether it matches a specific pattern. Non-compliant data is set to null for subsequent processing. Dirty data, such as variations in representing null values (null, Null, NULL, unknown), is also cleaned.

6. Data Preprocessing

After assessing data quality, it is necessary to handle abnormal data, primarily involving missing values and outliers.

6.1 Data Aggregation

Aggregating data based on month and 'uaid' (unique advertising identifier) to improve data value rates.

6.2 Missing Value Handling

Common missing value handling methods include deleting records with very few missing values, filling categorical data with mode or other classes, filling continuous value data with median, mean, or using interpolation methods.

6.3 Outlier Handling

Outlier detection methods include statistical analysis, density-based analysis, and seeking business support for determining normal sample values. Outlier handling methods include deleting records with few outliers or replacing outlier values with boundary values.

6.4 Data Filtering

Filter the data based on the percentile threshold of the number of non-empty fields. Remove any data that have a lower rate of non-empty columns. In this example, I have 353192539 records, so the amount of data is sufficient, then I selected 24 as the threshold. Any data with less than 24 non-empty columns will be filtered out. Even though that means only 1% of data will remain, I still have over 3 million data.
